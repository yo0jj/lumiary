## 1. 제품 개요

### 한 줄 정의
AI가 매일 전화를 걸어 오늘 하루를 물어보고, 대화를 요약해 기록으로 남겨주는 자기 탐구 서비스

### 네이밍
**Lumiary** = *Luminary* (빛을 발하는 것) + *Diary* (일기)  
"나를 비추는 대화형 일기" — 자기 인식이 높아질수록 더 밝아진다는 의미.

### 프로젝트 배경
서울대 디자인학부 졸업작품 **Lumiary** (책 형태 Physical Device + Mobile App)를 사이드 프로젝트로 발전시킨 것. 졸업작품에서 검증한 핵심 컨셉 — *"일상 속 가벼운 성찰 루틴 메이커"* — 를 유지하되, 디바이스 없이 **AI 전화**만으로 MVP를 구현한다.

### 핵심 전략
> "재미로 시작한 기록이 습관이 되어 일상 속 실천으로 이어지는 일방향적 흐름을 만든다."

글쓰기는 부담스럽지만 전화통화는 자연스럽다는 인사이트를 기반으로, **기록의 진입장벽을 낮추고 습관화**를 유도한다.

### 윤리적 한계
이 서비스는 전문적인 심리상담/치료를 대체하지 않는다. 일상에서 누구나 부담 없이 시작할 수 있는 자기 성찰의 첫걸음을 제공하는 도구다.

### 해결하는 문제

**배경: 자기 탐구 문화의 역설**  
MZ세대를 중심으로 MBTI, 퍼스널컬러, 심리테스트 등 자기 탐구 문화가 폭발적으로 성장했지만, 단편적 정보 제공에 그쳐 **깊이 있는 자기 이해로 이어지지 않는다.**

| 사용자 고통 | 현재 행동 | Lumiary의 해결 |
|---|---|---|
| 매일 기록하고 싶지만 작성이 귀찮다 | 다이어리 앱 설치 → 2~3일 후 포기 | AI가 먼저 전화를 건다 → 수동적 참여 |
| 쓰면 솔직하지 못하다 | 꾸미거나 의식하게 됨 | 대화는 즉흥적이라 솔직해짐 |
| 뭘 기록해야 할지 모른다 | 빈 화면 앞에서 막막함 | AI가 자유연상 방식으로 질문을 이끌어 흐름 제공 |
| 기록이 쌓여도 돌아보지 않는다 | 아카이브만 됨 | 대화 요약이 자동 생성되어 보기 쉬움 |
| 상담은 비용·심리적 장벽이 높다 | 혼자 고민, 머릿속이 정리 안 됨 | 매일 5~10분, 부담 없이 이야기 |
| AI 챗봇은 맥락 없이 대화만 이어진다 | 쓰다 지쳐서 포기 | 전화로 축적된 기록을 기반으로 점점 깊어지는 대화 |

## 2. 타겟 사용자

### 주요 페르소나
**"나를 스스로 관찰하고 분석해보고 싶은 욕구가 있는 10~30대"**

- 삶의 선택 앞에서 불안함을 느끼며 방향을 찾고 싶은 사람
- 기록을 좋아하지만 지속하는 것이 어려운 사람
- MBTI, 심리테스트 등 자기 탐구 콘텐츠를 즐겨 소비하지만 단편적이라 느끼는 사람
- 상담에 관심 있지만 비용/심리적 장벽으로 망설이는 사람

### 핵심 불편함 (인터뷰 기반)
실제 심층 인터뷰 5명에서 나온 인사이트:

1. **정리의 어려움**: "머릿속 생각이 너무 많아 복잡한데, 이를 정리해주는 존재가 있으면 좋겠다"
2. **시작의 막막함**: "일기는 쓰고 싶지만 어디서 시작해야 할지 모르겠다"
3. **지속의 어려움**: "꾸준히 지속하게 만드는 동기부여 장치가 필요하다"
4. **패턴 인식 욕구**: "자기 인식 데이터가 쌓이면 패턴과 인사이트를 보고 싶다"

## 3. AI 대화 방식 (핵심 설계 원칙)

단순 "오늘 하루 어땠어요?" 수준이 아니라, 심리학적 접근을 기반으로 깊이 있는 자기 탐구를 유도한다.

### 정신분석적 접근: 자유연상 기법
사용자가 마음속에 떠오르는 생각을 자유롭게 표현하도록 유도. 무의식 속 억압된 감정과 갈등을 탐색.

> "지금 떠오르는 게 뭐든 편하게 이야기해봐요."  
> "방금 말씀하신 부분에서 특별히 신경 쓰이는 게 있나요?"  
> "그 감정이 과거의 어떤 경험과 연결되는 것 같나요?"

### 인지행동치료적 접근: 소크라테스식 문답법
인지 왜곡 없이 자신을 파악하도록 돕는 질문. 생각을 단정하지 않고 탐색하게 유도.

> "그렇게 생각하게 된 근거가 뭐라고 느끼세요?"  
> "반대로 생각해보면 어떨까요?"

### 대화 구조 (매 통화)
1. **오프닝** (30초): 친근한 인사, 오늘의 분위기 파악
2. **자유 탐색** (3~6분): 사용자가 꺼내는 주제 따라가며 자유연상 유도
3. **소크라테스 질문** (1~2분): 한 가지 생각을 더 깊이 파고들기
4. **클로징** (30초): 오늘의 한 마디 + 다음 전화 예고

## 4. 핵심 기능 (MVP 스코프)

### In Scope

#### 4-1. 온보딩
- 이름 입력
- 전화 받고 싶은 시간대 설정 (ex. 매일 밤 10시)
- 전화번호 입력 (수신용)

#### 4-2. 홈 화면
- 오늘의 상태: "오늘 전화 예정 시간 OO:OO" / "오늘 대화 완료"
- 최근 대화 요약 카드 (최근 1개)
- 다음 전화까지 남은 시간 표시

#### 4-3. AI 전화 수신 → 대화 진행
- 설정한 시간에 AI가 아웃바운드 전화를 검
- 위의 대화 구조(오프닝 → 자유탐색 → 소크라테스 질문 → 클로징) 적용
- 통화 시간: 5~10분 목표

#### 4-4. 대화 요약
- 통화 종료 후 앱에 자동 생성
- 포함 내용:
  - 오늘의 키워드 (3개)
  - 한 줄 요약
  - 대화 전문 (접기/펼치기)
- 요약 생성 후 푸시 알림

#### 4-5. 만족도 평가
- 요약 하단에 별점 1~5점
- 선택적 코멘트 입력

#### 4-6. 마이 페이지
- **대화 기록**: 날짜별 요약 리스트
- **설정**: 전화 시간 변경, 이름 변경, 전화번호 변경

### Out of Scope (v2 이후)

| 기능 | 이유 |
|---|---|
| Physical Device (책 형태 디바이스) | 졸업작품의 핵심 컨셉이지만 하드웨어 개발 기간 필요 |
| 인사이트/패턴 분석 | 데이터 3개 이상 축적 후 제공 — 먼저 기록 습관 검증 필요 |
| 체크인 (감정 기록, 사진) | 앱 내 능동적 입력 — 전화 기반 수동적 참여 가설 검증 먼저 |
| AI 채팅 (텍스트) | 전화 가설 검증 후 보완 채널로 추가 |
| 소셜 기능 (공유, 팔로우) | — |
| 구독 결제 | 무료로 초기 사용자 확보 우선 |
| AI 페르소나 선택 | — |

## 5. 유저 플로우

```
[최초 진입]
앱 설치 → 온보딩 (이름 + 전화번호 + 선호시간) → 홈 화면

[매일 반복]
설정 시간 → AI 아웃바운드 전화 → 사용자 수신
→ 대화 진행 (5~10분) → 통화 종료
→ 앱에서 대화 요약 생성 (푸시 알림)
→ 사용자가 요약 확인 → 만족도 평가

[언제든지]
홈 → 대화 기록 → 과거 요약 열람
마이 → 설정 변경
```

## 6. 성공 기준

### 초기 가설 검증 지표

| 가설 | 검증 지표 | 목표 |
|---|---|---|
| 전화를 걸어주면 꾸준한 기록이 가능하다 | **7일 연속 통화율** | > 40% |
| 전화는 거부감 없이 솔직하게 말할 수 있다 | **평균 통화 시간** | > 3분 |
| 기록이 쌓이면 가치를 느낀다 | **요약 열람률** (통화 후 24시간 내) | > 70% |

### 보조 지표
| 지표 | 목표 |
|---|---|
| 온보딩 완료율 | > 80% |
| 수신율 (전화 왔을 때 받는 비율) | > 60% |
| 만족도 평균 별점 | > 4.0 / 5.0 |
| 3회 이상 통화한 사용자 비율 | > 50% |

### 실패 신호
- 수신율 < 40% → 전화 시간대 / 알림 방식 재설계 필요
- 평균 통화시간 < 90초 → 대화 시나리오 개선 필요
- 7일 연속 통화율 < 20% → 핵심 가설 재검토

## 7. 기술 스택 (1주일 MVP 기준)

| 영역 | 선택 | 이유 |
|---|---|---|
| **플랫폼** | 웹앱 (모바일 반응형) | 앱 심사 없이 즉시 배포 가능 |
| **프론트엔드** | Next.js (React) | 빠른 개발, 배포 간편 |
| **백엔드** | FastAPI (Python) | AI 연동이 쉽고 빠름 |
| **데이터베이스** | Supabase (PostgreSQL) | Auth + DB 한번에 해결 |
| **AI 대화** | Claude API (claude-sonnet-4-6) | 자연스러운 한국어 대화, 자유연상/소크라테스 프롬프팅에 적합 |
| **STT** | Deepgram | 실시간 한국어 음성인식 |
| **TTS** | ElevenLabs 또는 Clova Voice | 자연스러운 한국어 TTS |
| **아웃바운드 전화** | Twilio Programmable Voice | 전화 발신 + 통화 제어 |
| **배포** | Vercel (FE) + Railway (BE) | 무료 티어, 빠른 배포 |
| **푸시 알림** | Firebase Cloud Messaging (FCM) | 무료, 웹 푸시 지원 |

### 개발 리스크 관리
- **Twilio 한국 번호 발신**: 한국 발신 번호 등록에 1~2일 소요될 수 있음 → 미국 번호로 먼저 테스트
- **STT 지연**: 실시간 처리 지연 시 WebSocket 연결 유지 필요
- **통화 품질**: MVP는 완벽한 자연어보다 "동작하는 흐름" 우선
